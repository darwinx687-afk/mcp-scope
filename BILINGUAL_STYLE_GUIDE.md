# Bilingual Style Guide

## English

- Use clear, honest, technical language.
- Avoid hype.
- Prefer concrete nouns: report, warning, risk note, capability category, local config.
- Do not imply production readiness before the project earns it.
- Do not claim complete protection.

## Chinese

- 语气自然、直接、偏开发者表达。
- 不要写成营销稿。
- 保留清楚的边界：本地优先、报告优先、不执行 MCP 服务、不请求实时工具元数据。
- 不要制造用户数量、收入、下载量等虚假势能。
- 安全相关输出应称为“风险提示”“信号”或“警告”，不要说成绝对证明。

## Banned Chinese Phrases

- 赋能
- 闭环
- 矩阵
- 生态
- 革新
- 企业级

## Security Language

Use:

- risk note
- signal
- warning
- transparency report
- review evidence

Avoid:

- proven safe
- complete protection
- guaranteed detection
- production-grade scanner
- exploit-ready
