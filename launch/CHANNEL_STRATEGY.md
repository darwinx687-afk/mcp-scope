# Launch Channel Strategy

All publishing remains manual. Do not publish the GitHub prerelease, social posts, npm package, or GitHub Marketplace listing from automation.

## Selected First-Wave Channels

Chinese:

1. Juejin
2. Xiaohongshu
3. Jike
4. WeChat group
5. WeChat moments

English:

1. GitHub prerelease
2. LinkedIn
3. X / Twitter
4. Hacker News / Show HN
5. Dev.to

Optional later:

- Reddit, only after explicit human approval.

V2EX is skipped for this launch wave.

## Recommended Posting Order

1. Publish the GitHub prerelease manually after final review.
2. Post the Juejin technical article with a Chinese banner/card.
3. Post the LinkedIn builder update with an English card.
4. Post the X / Twitter short post or thread with an English card.
5. Post Xiaohongshu with the Chinese square card and less technical wording.
6. Share in WeChat group, WeChat moments, and Jike with Chinese cards.
7. Post Hacker News Show HN only after the GitHub page and README are manually checked.
8. Publish the Dev.to technical article after LinkedIn/X or as long-tail content.

## Why V2EX Is Skipped

V2EX is skipped for this first wave to keep the launch focused on channels with clearer fit for this preview: Juejin for technical depth, Xiaohongshu/Jike/WeChat for Chinese reach, and GitHub/LinkedIn/X/HN/Dev.to for English developer visibility.

Keep `launch/copy/v2ex.md` as an archive, but do not post it.

## Why Reddit Is Optional Later

Reddit can be useful but varies heavily by subreddit rules and moderation expectations. Treat it as optional later work, and only post after explicit human approval and subreddit-specific review.

## Why Hacker News Is High-Reach But Strict

Hacker News can reach many developers quickly, but Show HN posts are judged sharply on clarity, novelty, and whether the project is genuinely ready to inspect. Post there only after the GitHub prerelease is public and the README has been manually checked.

## First Post

The first public step should be the GitHub prerelease. After that, the safest first public posts are Juejin and LinkedIn.

## Do Not Automate

Do not post automatically. Preview manually before publishing. Do not use personal photos, random album photos, external images, remote images, or generated claims about npm, Marketplace, official integrations, mature security protection, or traction.
