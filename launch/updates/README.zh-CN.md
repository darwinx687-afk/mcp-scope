# 更新发布流程

这个目录存放 MCP Scope 的版本化更新发布材料。这里的文件只是草稿和检查清单，不会发布 release、创建 tag、发社交平台、发布 npm 或发布 GitHub Marketplace。

## 当前更新包

- [v0.2.0-preview](v0.2.0-preview/)：双语入口与使用体验完善版更新包。

## 规则

- 发布前先人工审查 release notes。
- 社交平台内容必须先预览再发布。
- 人工发布后，把最终 URL 记录到对应版本的 posting tracker。
- 在真实发布前，不要写 npm、Marketplace、官方集成或生产级安全保护声明。
