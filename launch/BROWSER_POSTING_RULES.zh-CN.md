# 浏览器发布规则

这些规则适用于未来任何手动或浏览器辅助发布。

- 所有发布都必须人工执行。
- 中文平台必须使用中文文案和 zh-CN 图片。
- 海外平台必须使用英文文案和 en 图片。
- 只使用 `launch/assets/` 中指定的本地 SVG 源文件，或 `launch/assets/exports/` 中的 PNG 导出文件。
- 不选择个人照片。
- 不选择随机相册图片。
- 不使用外部图片或远程图片 URL。
- V2EX 不发布。
- Hacker News 默认纯文字，除非人工明确决定使用图片。
- 如果以后使用浏览器自动化，Reddit 和 Hacker News 发布前必须有明确人工批准。
- 遇到登录时暂停，交给人手动处理。
- 遇到 captcha、2FA 或安全检查时暂停。
- 不保存密码。
- 不绕过平台规则。
- 发布前先预览。
- 把发布链接记录到 `launch/POSTING_TRACKER.md`。
- 失败要如实记录。
- 失败后不要反复刷屏重试。
- 没有明确确认时，不通过浏览器自动把 GitHub release 转为正式发布。
