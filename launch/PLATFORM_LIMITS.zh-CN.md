# 平台限制和表达方式

发布前先检查平台当前限制。字数、图片和格式规则会变化，不要靠旧印象。

## 小红书

使用中文社交卡片。写得像一个自然的项目记录，说明为什么做它。不要宣传感太重，不用个人照片，也不要用随机相册图片。

## 掘金

按技术文章写。包含背景、命令、报告示例、局限和 GitHub 链接。

## V2EX

简洁、少宣传感。说清楚做了什么，向正在用 MCP 或 AI coding 工具的人请教反馈。

## LinkedIn

用 builder update 的方式。重点讲问题、本地优先、预览范围和希望收到反馈。

## X / Twitter

可以用一条短帖，也可以用短 thread。需要配图时使用英文社交卡片。

## Reddit 和 Hacker News

只有明确确认后再发。语气简单直接，以社区反馈为主。

## 微信

微信群更短、更口语。朋友圈可以偏个人记录。只使用准备好的 SVG 图片路径。
