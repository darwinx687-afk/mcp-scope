## What Not To Do Yet

- Do not add cloud features because one person asks.
- Do not implement live execution without explicit consent design.
- Do not claim production-grade protection.
- Do not add npm or Marketplace packaging until maintainers choose that phase.

## Decision Rules

- Fix docs first if confusion repeats.
- Fix false positives before adding new features.
- Add config shapes only if multiple users ask or evidence is strong.
- Prefer small local improvements over broad platform features.
